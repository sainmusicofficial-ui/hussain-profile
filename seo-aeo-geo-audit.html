Issue Details

01 Contact page had copy-pasted Work metadata SEO  ·  Critical  ·  contact/layout.jsx
Problem
The contact page exported title: "Work", a description about case studies, and a canonical pointing to /work. Google would have indexed the Contact page as a near-duplicate of the Work page, suppressing it from rankings entirely.
Fix
Replaced with correct Contact metadata: title "Contact", description about hiring and getting in touch, canonical correctly set to /contact, function renamed from WorkLayout to ContactLayout.
02 Canonical URL inconsistency (www vs non-www) SEO  ·  High  ·  app/layout.js
Problem
metadataBase was set to hussainkhan.co.in (no www) but the root canonical was set to www.hussainkhan.co.in. Google can treat these as separate origins and split PageRank between them instead of consolidating it.
Fix
Standardised to hussainkhan.co.in (non-www) everywhere, consistent with the metadataBase value. Also fixed the indentation formatting of the alternates block.
03 Case study pages had zero unique metadata SEO  ·  High  ·  work/[slug]/layout.jsx (new)
Problem
Because page.jsx uses "use client", Next.js cannot export generateMetadata from it. Every case study page — medizap, ruyadar, brisket-and-chops, ss-auditors, olesome, aurenza, myas-and-co — was serving identical metadata inherited from the root layout. Google saw 7 identical pages with the same title and description.
Fix
Created src/app/work/[slug]/layout.jsx as a server component that runs generateMetadata({ params }), pulling each project's title, description, and services from caseStudies.js. Every project now gets a unique title, description, canonical, OpenGraph, and Twitter card.
04 Root metadata — weak keywords, no location targeting, no googleBot directives SEO / GEO  ·  Medium  ·  app/layout.js
Problem
Keywords were generic with no city or country context. No googleBot directive meant Google could truncate snippets and preview images by default. The OG image was not declared as a proper object so social shares showed no preview. Locale was en_US despite the site being India-based. No creator or category fields for AI entity recognition.
Fix
Added location-specific keywords (Brand Designer Bangalore, Freelance Brand Designer India, Logo Designer Bangalore, etc.). Added explicit googleBot object with max-image-preview: large and max-snippet: -1 to allow rich snippets. Added creator, category fields. Declared full OG image object (1200×630). Changed locale to en_IN.
05 Missing FAQPage, WebSite, and AggregateRating schemas AEO / GEO  ·  Critical  ·  components/JsonLd.jsx
Problem
The existing JSON-LD only had Person and ProfessionalService schemas. This meant:
  • No FAQ rich snippets in Google Search results
  • No star ratings from testimonials showing in search
  • AI tools (ChatGPT, Perplexity, Google SGE) had no structured Q&A to cite when someone asks "who is Hussain Khan" or "find a brand designer in Bangalore"
  • No WebSite entity for Google Knowledge Panel
⚠ Reverted
Four schemas were written and applied but were reverted by a linter. They need to be re-committed. See the Pending Action section below for what exactly needs to go back in.

Pending Action — Re-apply JsonLd.jsx

Action required: The JSON-LD upgrades were written during this session but were reverted. The file src/components/JsonLd.jsx needs to be updated and committed before the linter runs.

The four schemas that need to go into JsonLd.jsx:

1. FAQPage schema

8 Q&As covering the most common search queries:

  • What services does Hussain Khan offer?
  • Where is Hussain Khan based?
  • How long does brand identity design take? (3–6 weeks)
  • How long does website design take? (4–8 weeks)
  • How long does startup MVP design take? (2–4 weeks)
  • What tools does he use? (Figma, React, Next.js, etc.)
  • How many projects completed? (50+, 30+ clients, 4+ years)
  • How do I contact Hussain Khan?

2. AggregateRating + Review schema

5/5 rating built from the three existing testimonials (Shuaib / Medizap, Affan Khan / Ruyadar, Sayeed / SS Auditors). Enables star ratings in Google search results — a significant click-through rate boost.

3. WebSite schema with SearchAction

Declares the site as a searchable entity. Used by Google to build a Knowledge Panel and by AI models to understand the site's purpose and author.

4. Enriched Person + ProfessionalService with @id cross-references

Each schema gets a stable @id (e.g. https://hussainkhan.co.in/#person) so Google and AI models can resolve them as the same entity across pages.
